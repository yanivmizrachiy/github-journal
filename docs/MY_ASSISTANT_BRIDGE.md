# MY ASSISTANT BRIDGE

## מטרה
לחבר את `my-assistant` ל-`github-journal` כך שכל שמירה אמיתית תעבור דרך GitHub בצורה מבוקרת וללא דמו.

## עקרונות
- `github-journal` הוא מקור האמת של היומן והנתונים.
- `my-assistant` משמש כמרכז אוטומציות וביצוע.
- כל שמירה חייבת להסתיים בשינוי אמיתי בריפו וב-commit אמיתי.
- אין כפתורי שמירה מזויפים.

## זרימת שמירה אמיתית
1. המשתמש עורך יום ביומן או שיעור פרטי.
2. האפליקציה מייצרת payload JSON תקין.
3. נוצר Issue/patch דרך מנגנון `my-assistant`.
4. Workflow של `my-assistant` או של `github-journal` מעדכן את קובץ היעד.
5. נוצר commit אמיתי.
6. ניתן לאמת את השינוי בריפו.

## קבצי יעד
- `entries/YYYY-MM-DD.json`
- `data/private-lessons.json`
- `RULES.md`

## דרישות אימות
- כל שינוי חייב להיות ניתן לאימות דרך commit.
- כל כפתור שמור חייב לשנות קובץ אמיתי.
- כל דרישה חדשה של המשתמש חייבת להיכנס גם ל-`RULES.md`.

## מצב נוכחי
- קיימים דפי תצוגה ועריכה.
- קיימים קבצי נתונים בסיסיים.
- השלב הבא הוא חיבור שמירה אמיתית דרך `my-assistant`.
