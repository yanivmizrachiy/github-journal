# SAVE FLOW

## מטרה
לאפשר שמירה אמיתית של יום ביומן בלי דמו.

## עקרונות
- GitHub הוא מקור האמת
- כל יום נשמר בקובץ `entries/YYYY-MM-DD.json`
- אין כפתורי שמירה מזויפים
- כל שינוי חייב להיות מתועד

## מבנה השמירה המתוכנן
1. בחירת יום
2. טעינת תוכן קיים
3. עדכון אירועים / הערות / צבעים
4. יצירת payload תקין
5. כתיבה אמיתית לקובץ היום
6. עדכון `updated_at`
7. תיעוד שינוי

## שדות חובה ליום
- day_name
- date_gregorian
- date_hebrew
- events
- notes
- colors
- updated_at

## כיוון היישום
הדף `editor.html` מייצר payload מוכן. השלב הבא הוא חיבור שמירה אמיתית לריפו.
